# AutoDL

영상 **다운로드**와 **화면 영역 녹화**를 한 창에서 처리하는 데스크톱 앱.

- **다운로드 탭**: URL만 붙여넣으면 [yt-dlp](https://github.com/yt-dlp/yt-dlp)(유튜브·트위터 등
  수천 개 사이트)로 **가능한 최고화질** 영상+음성을 받아 MP4로 합칩니다.
- **화면 영역 탭**: 화면에서 원하는 영역만 드래그로 골라 MP4로 녹화합니다(ffmpeg gdigrab).
- **창 녹화 탭**: 특정 창을 **다른 창이 위를 덮어도** 그 창 내용만 녹화합니다.
  OBS와 같은 Windows Graphics Capture(WGC)를 사용해 GPU 가속 브라우저(Chrome·Edge·
  Whale 등)도 검은 화면 없이 잡힙니다. 창의 일부 영역만 잘라 녹화하는 것도 가능합니다.

## 설치

```bash
pip install -r requirements.txt
```

> ffmpeg는 시스템에 설치돼 있으면 그걸 쓰고, 없으면 `imageio-ffmpeg`(pip로 함께 설치됨)에
> 포함된 빌드를 자동으로 사용합니다. 별도 설치가 필요 없어요.

## 실행

```bash
python autodl.py
```

**다운로드:** URL 붙여넣기 → 저장 폴더 확인 → **⬇ 최고화질로 다운로드**

**화면 영역 녹화:** `● 화면 영역` 탭 → **영역 선택**(화면을 드래그) → **● 녹화 시작** → **■ 녹화 중지**
- 화면의 고정된 좌표를 찍으므로, 그 위로 다른 창이 올라오면 그 창이 찍힙니다.

**창 녹화 (다른 창이 가려도 OK):** `● 창 녹화` 탭 → 목록에서 **창 선택** → (선택) `창의 일부만 녹화` 체크 후
**영역 선택** → **● 녹화 시작** → **■ 녹화 중지**
- 다른 창이 위를 덮거나 다른 앱으로 전환해도 그 창 내용만 계속 녹화됩니다.
- 창을 **최소화하면** 캡처가 멈춥니다(화면에서 사라지므로).

> 소리 녹음: **영역 탭**은 시스템 전체 소리, **창 녹화 탭**은 기본으로 **그 창의 앱 소리만** 녹음합니다.

### 창 녹화의 소리 옵션

창 녹화 탭의 `소리` 드롭다운에서 고를 수 있습니다.

| 옵션 | 동작 |
| --- | --- |
| **이 창 앱 소리만 (기본)** | 그 창을 띄운 앱의 소리만 녹음. 다른 앱·알림 소리는 제외 (WASAPI 프로세스 루프백). 실패하면 자동으로 전체 소리로 폴백. |
| **전체 시스템 소리** | 스피커로 나가는 모든 소리 |
| **소리 없음** | 영상만 |

> 브라우저는 모든 탭이 한 프로세스라 **"그 브라우저 전체 소리"** 가 녹음됩니다(탭 하나만 분리는 불가).
> 또한 음소거하거나 볼륨 0이면 재생되는 소리가 없어 녹음에도 안 들어갑니다.

### 녹화 옵션 (영역·창 탭 공통)

| 옵션 | 설명 |
| --- | --- |
| **화질** | 표준(CRF 23) · **고화질(CRF 18, 기본)** · 최고화질(CRF 14). 숫자가 낮을수록 더 선명하고 용량이 큽니다. 해상도는 항상 원본 그대로입니다. |
| **마우스 커서 표시** | 기본 **꺼짐**(커서가 녹화에 안 보임). 켜면 커서가 함께 찍힙니다. |
| **타이머** | 분·초 지정 시 그 시간이 지나면 자동으로 녹화가 멈춥니다. `0:0`이면 수동 종료. |
| **완료 알림** | 타이머로 끝나면 소리와 함께 알림 창이 뜹니다. |

### 가렸을 때 흰 화면이 나온다면 (브라우저)

Chrome·Edge·Whale 같은 크로미움 브라우저는 창이 **가려지면 절전을 위해 그리기를 멈춥니다.**
그러면 녹화 영상이 흰/멈춘 화면이 됩니다. 창 녹화 탭의 **`가려도 녹화되게 Whale 실행`** 버튼을
쓰거나, 브라우저를 아래 플래그로 직접 실행하세요(기존 창은 모두 닫은 뒤):

```
whale.exe --disable-features=CalculateNativeWinOcclusion --disable-backgrounding-occluded-windows
```

### YouTube 최상위 화질 (선택)

최신 yt-dlp는 일부 YouTube 영상의 최고화질 포맷을 뽑을 때 JS 런타임([Deno](https://deno.com))을
요구합니다. 없어도 대부분 4K까지 받아지지만, 간혹 최상위 포맷이 누락될 수 있어요.
최대치를 보장하려면 Deno를 설치하세요(Windows): `winget install DenoLand.Deno`

## 동작 방식

- 화질: `bv*+ba/b` — 받을 수 있는 최고 화질 영상 + 최고 음질 오디오를 따로 받아 합칩니다.
- 합치기: 재인코딩 없이 컨테이너만 MP4로 바꾸므로 **화질 손실이 없습니다**.
  (MP4에 담을 수 없는 코덱이면 원본 컨테이너를 유지)

## 파일 구성

| 파일 | 역할 |
| --- | --- |
| `autodl.py` | tkinter GUI (다운로드 + 영역 녹화 + 창 녹화 탭, 영역 선택 오버레이) |
| `downloader.py` | yt-dlp 래퍼(다운로드 엔진) |
| `recorder.py` | 녹화 엔진: `RegionRecorder`(gdigrab 영역) + `WindowRecorder`(WGC 창), 오디오 합치기 |
| `audiocap.py` | 시스템 전체 소리 루프백 녹음(WASAPI) |
| `procaudio.py` | 특정 앱(프로세스) 소리만 녹음(WASAPI 프로세스 루프백, COM) |
| `winutil.py` | 창 열거·좌표·DPI·PID·브라우저 실행 헬퍼 (ctypes) |
| `requirements.txt` | 의존성 |
